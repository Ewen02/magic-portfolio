## Vue d'ensemble

**Swipe Movie** résout un problème universel : passer des heures à décider quoi regarder ensemble.

Chaque participant swipe (like/dislike) sur des films, et quand tout le monde aime le même film, c'est un **match** ! Grâce aux rooms avec codes de partage, les utilisateurs peuvent rejoindre une session même sans compte.

## Fonctionnalités principales

### Core
- Système de **swipe** avec animations fluides
- **Rooms** avec codes de partage (join sans compte)
- **Matching en temps réel** via WebSocket
- Support **Films & Séries TV**
- **PWA** installable sur mobile

### Filtres avancés
- Genres (20+)
- Note minimum, année de sortie, durée
- **Plateformes de streaming** (50+ providers)
- Langue originale, région de disponibilité

### Intégrations
- **TMDB** : catalogue de 10,000+ films/séries
- **Trakt.tv** : sync historique de visionnage (OAuth)
- **AniList** : sync bibliothèque anime (OAuth)
- **Stripe** : paiements et abonnements

### Plans

|---|---|---|
| Rooms actives | 2 | Illimité |
| Participants/room | 5 | 20 |
| Swipes/mois | 100 | Illimité |
| Filtres avancés | Limités | Tous |

## Architecture Monorepo

```
swipe-movie/
├── apps/
│   ├── web/          # Next.js 16 (App Router)
│   └── api/          # NestJS 11
├── packages/
│   ├── database/     # Prisma schema + client
│   ├── ui/           # Design system (Atomic Design)
│   ├── types/        # Types partagés
│   ├── email/        # Service email (Resend)
│   └── subscription/ # Logique plans FREE/PRO
└── turbo.json
```

## Stack technique

### Frontend
- **Framework** : Next.js 16, App Router, Turbopack
- **UI** : Tailwind CSS 4, Radix UI, Framer Motion
- **State** : SWR
- **Auth** : Better Auth (Google OAuth)
- **Real-time** : Socket.IO
- **i18n** : next-intl (FR/EN)
- **PWA** : Serwist
- **Monitoring** : Sentry, Vercel Analytics

### Backend
- **Framework** : NestJS 11
- **Database** : PostgreSQL + Prisma
- **Cache** : Redis
- **Real-time** : Socket.IO (WebSocket)
- **Auth** : JWT + Passport
- **Payments** : Stripe webhooks
- **Docs** : Swagger

### DevOps
- **Monorepo** : Turborepo + pnpm
- **CI/CD** : GitHub Actions
- **Deploy** : Vercel (frontend), Railway (backend)

## Défis et enseignements

- Concevoir une **architecture monorepo scalable** avec Turborepo
- Implémenter un **matching temps réel** performant avec WebSocket et Redis
- Gérer la **synchronisation d'état** entre plusieurs clients dans une room
- Intégrer des **APIs OAuth tierces** (Trakt.tv, AniList)
- Mettre en place un **système de plans** flexible avec Stripe
- Optimiser les **performances du swipe** (animations, prefetching)
